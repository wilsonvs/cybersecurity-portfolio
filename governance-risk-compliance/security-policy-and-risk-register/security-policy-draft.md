# Security Policy Draft

## Purpose

I wrote this short policy draft to practice converting security expectations into clear rules that users and system owners can understand.

## Scope

This sample policy applies to users, endpoints, cloud accounts, and systems used in the lab scenario.

## Policy Statements

1. Accounts must use unique user IDs and should not be shared.
2. MFA must be enabled for administrator accounts and remote access.
3. Systems must receive security updates within the approved patch window.
4. Suspicious emails, login prompts, or security alerts must be reported.
5. Logs for critical systems must be retained long enough to support investigation.
6. Privileged access must be reviewed on a recurring schedule.

## Evidence I Would Collect

| Control Area | Evidence |
| --- | --- |
| MFA | MFA policy export or screenshot |
| Patch management | Patch report or vulnerability scan summary |
| Logging | SIEM retention setting and alert evidence |
| Access review | Access review signoff and exception list |
| Incident response | Incident ticket, report, and timeline |
