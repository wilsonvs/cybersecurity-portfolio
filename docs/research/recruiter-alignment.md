# Recruiter Alignment Research

## Purpose

This document explains why the portfolio focuses on SOC analysis, SIEM/log analysis, incident response, vulnerability management, Python automation, and GRC. These areas repeatedly appear in entry-level cybersecurity analyst and SOC analyst role expectations.

## Research Summary

| Hiring Signal | Portfolio Evidence |
| --- | --- |
| Security monitoring and alert triage | SOC Home Lab and credential attack case study |
| SIEM and log analysis | SIEM Detection Lab, SPL/KQL examples, Windows Event Log Analysis |
| Incident response documentation | Incident report case study and incident response template |
| Vulnerability management | Vulnerability Assessment Report and remediation plan |
| Python automation | Python Log Triage Tool and sample authentication events |
| GRC and risk communication | Security Policy and Risk Register project |
| MITRE ATT&CK knowledge | Detection writeups and case study mappings |

## Portfolio Strategy

Recruiters and hiring managers need fast evidence that a candidate can do realistic junior analyst work. This portfolio is organized to show:

1. I understand what security alerts mean.
2. I can investigate using logs and context.
3. I can write clear incident notes.
4. I can explain risk to technical and non-technical audiences.
5. I can automate simple repetitive analyst tasks.
6. I can map findings to recognized frameworks such as MITRE ATT&CK, NIST, and GRC practices.
