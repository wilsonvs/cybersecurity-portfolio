# Security Policy

## Scope

This repository contains portfolio documentation and learning artifacts. It should not contain production secrets, private credentials, customer data, or sensitive personal information.

## Reporting a Security Issue

If you notice a security concern in this repository, open a GitHub issue with a clear description of the problem and the affected file or workflow.

## Handling Sensitive Data

Do not commit:

- API keys
- Passwords
- Private tokens
- Real customer data
- Private logs
- Sensitive screenshots

Use sanitized examples, generated sample data, or public lab data only.
