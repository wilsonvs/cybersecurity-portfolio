### Step 2: Import Process

1. **Login to Zoho Desk**
   - Navigate to Knowledge Base section

2. **Start Import**
   - Click on "Articles" or "Knowledge Base"
   - Look for "Import" or "Bulk Import" option
   - Select CSV import method

3. **Upload File**
   - Select: `cyberphore_knowledgebase_FINAL.csv`
   - Encoding: UTF-8

4. **Map Fields**
   ```
   CSV Column          →    Zoho Desk Field
   ────────────────────────────────────────
   Article Title       →    Title
   Slug                →    Permalink/SEO URL
   Category            →    Category
   Sub-Category        →    Sub-Category  
   Owner               →    Author/Owner
   Tags                →    Tags/Keywords
   Body (HTML)         →    Content/Article Body
   Meta Description    →    SEO Description/Summary
   ```

5. **Import Settings**
   - Status: Set to "Draft" initially (review before publishing)
   - Visibility: Internal or Public (per your preference)
   - Language: English
   - Version: Create new articles

6. **Execute Import**
   - Click "Import" or "Start Import"
   - Monitor progress
   - Check for any errors

## 🔍 Troubleshooting

### Issue: Import Fails

**Solutions:**
- Check CSV file encoding (must be UTF-8)
- Verify all required columns are present
- Ensure no special characters in category names
- Try importing in smaller batches (use the batch files)

### Issue: HTML Not Rendering

**Solutions:**
- Verify Zoho Desk allows HTML in article body
- Check if HTML needs to be in specific format
- Test with one article first
- Contact Zoho support for HTML requirements

### Issue: Images Not Showing

**Solutions:**
- Unsplash URLs should work automatically
- If blocked, download images and re-upload to Zoho
- Alternative: Use Zoho's image hosting
- Check if external images are allowed in your plan

### Issue: Categories Not Matching

**Solutions:**
- Pre-create all categories before import
- Ensure exact spelling matches CSV
- Use category mapping during import
- Manually assign categories after import if needed

### Issue: Owner "Wilson S" Not Found

**Solutions:**
- Create user "Wilson S" before import
- Map to different existing user during import
- Assign owner after import
- Use your admin account temporarily
